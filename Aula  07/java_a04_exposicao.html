Switch Avançado
e Controle com Laços

Feche o módulo de Controle de Fluxo dominando o switch com String e enum, a nova sintaxe de switch expression do Java 14+, e as técnicas de break e continue com rótulos para laços aninhados.

Switch com String

Comparando texto no switch

Desde o Java 7, o switch aceita String como expressão — além de int, char e enum. Internamente usa .equals(), por isso é case-sensitive: "sim" e "SIM" são cases diferentes.

JAVA — switch com String
Scanner sc = new Scanner(System.in);
System.out.print("Dia da semana (seg, ter, qua...): ");
String dia = sc.next().toLowerCase(); // normaliza para minúsculo

switch (dia) {
    case "seg": case "segunda":
        System.out.println("Segunda-feira — início da semana!");
        break;
    case "sex": case "sexta":
        System.out.println("Sexta-feira — quase lá!");
        break;
    case "sab": case "sábado":
    case "dom": case "domingo":
        System.out.println("Fim de semana! 🎉");
        break;
    default:
        System.out.println("Dia não reconhecido: " + dia);
}
💡
toLowerCase() antes do switch: normalizar a entrada com .toLowerCase() ou .trim() antes do switch é uma boa prática — evita duplicar cases para "Seg", "SEG", "seg" etc.
Switch Expression Java 14+

A nova sintaxe — switch como expressão

O Java 14 introduziu o switch expression com a seta →. Ele retorna um valor, elimina o break, e cada ramo é uma expressão (não um bloco com fall-through). É mais seguro e conciso que o switch clássico.

Switch clássico (Java 1–13)
Statement — executa ações
Requer break em cada case. Fall-through silencioso se esquecer. Não retorna valor diretamente.
Switch expression (Java 14+)
Expression — produz valor
Usa case X →. Sem fall-through. Pode ser atribuído diretamente a uma variável. Mais seguro.
JAVA — switch expression com →
int mes = 4;

// Switch expression: o resultado é ATRIBUÍDO à variável
String estacao = switch (mes) {
    case 12, 1, 2  -> "☀️ Verão";
    case 3, 4, 5  -> "🍂 Outono";
    case 6, 7, 8  -> "❄️ Inverno";
    case 9, 10, 11 -> "🌸 Primavera";
    default          -> "Mês inválido";
};
System.out.println("Estação: " + estacao);

// Múltiplas linhas no ramo: use yield para retornar o valor
int pontos = switch (mes) {
    case 1, 2 -> {
        System.out.println("Início do ano!");
        yield 10;   // yield substitui return dentro do switch
    }
    default -> 0;
};
🔢
Múltiplos valores em um case: case 12, 1, 2 -> agrupa valores separados por vírgula — substitui o fall-through intencional do switch clássico de forma explícita e segura.
Switch com Enum

enum — o parceiro natural do switch

O enum define um conjunto fixo de constantes nomeadas. O switch com enum é mais seguro que com int — o compilador avisa se algum case for omitido e não existe risco de valor inválido.

JAVA — enum + switch expression
// Definição do enum — pode ser em arquivo separado ou dentro da classe
enum Nivel { INICIANTE, INTERMEDIARIO, AVANCADO }

Nivel aluno = Nivel.INTERMEDIARIO;

// Switch expression com enum — sem default necessário se todos os casos forem cobertos
int cargaHoraria = switch (aluno) {
    case INICIANTE      -> 40;
    case INTERMEDIARIO   -> 60;
    case AVANCADO        -> 80;
};

System.out.printf("Nível %s: %d horas de curso%n", aluno, cargaHoraria);
💡
Enum é type-safe: ao contrário de um int que pode ter qualquer valor, um enum só aceita os valores definidos. O compilador rejeita atribuições inválidas em tempo de compilação — sem surpresas em tempo de execução.
Break e Continue com Rótulos

Saindo de laços aninhados com precisão

Em laços aninhados, break e continue afetam apenas o laço mais interno. Para sair de um laço externo diretamente, use um rótulo (label) — um identificador seguido de : antes do laço.

JAVA — labeled break
// Busca em matriz 3×3 — para imediatamente ao encontrar o valor
int[][] matriz = {{1,2,3},{4,5,6},{7,8,9}};
int alvo = 5;
boolean encontrado = false;

// O rótulo 'busca' identifica o laço externo
busca:
for (int i = 0; i < matriz.length; i++) {
    for (int j = 0; j < matriz[i].length; j++) {
        if (matriz[i][j] == alvo) {
            encontrado = true;
            System.out.printf("Encontrado em [%d][%d]%n", i, j);
            break busca;  // sai do laço 'busca' (o externo!)
        }
    }
}
System.out.println(encontrado ? "Busca encerrada." : "Não encontrado.");
JAVA — labeled continue
// Pula para a próxima iteração do laço EXTERNO
externo:
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        if (j == 2) {
            continue externo;  // pula j=2 e j=3 — vai direto para i+1
        }
        System.out.printf("i=%d j=%d%n", i, j);
    }
}
// Saída: i=1 j=1 | i=2 j=1 | i=3 j=1  (j=2 e j=3 nunca aparecem)
⚠️
Use rótulos com moderação: embora válidos, rótulos tornam o código difícil de seguir. Prefira refatorar para um método com return — o efeito é o mesmo e o código fica mais limpo. Rótulos são aceitáveis apenas em buscas em estruturas aninhadas sem métodos disponíveis.
Padrão de Saída Antecipada

return como controle de fluxo

Dentro de um método, return encerra a execução imediatamente. Usar return antecipado é mais limpo que rótulos e evita else profundamente aninhados — padrão chamado de guard clause.

JAVA — guard clause vs else aninhado
// ❌ Estilo com else aninhado — difícil de ler
static void processarIdade_ruim(int idade) {
    if (idade >= 0) {
        if (idade < 18) {
            System.out.println("Menor de idade");
        } else {
            System.out.println("Maior de idade");
        }
    } else {
        System.out.println("Idade inválida");
    }
}

// ✅ Guard clause — fail fast, caminho feliz no final
static void processarIdade(int idade) {
    if (idade < 0) {
        System.out.println("Idade inválida");
        return;           // saída antecipada
    }
    if (idade < 18) {
        System.out.println("Menor de idade");
        return;
    }
    System.out.println("Maior de idade");  // caminho feliz
}
